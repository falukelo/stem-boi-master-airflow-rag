# Course Blueprint: Mastering Workflow Orchestration

พิมพ์เขียวการเรียนการสอนแบบ Instructor-Led ระยะเวลา 1 วัน (6 ชั่วโมง) เพื่อเตรียมบุคลากรเข้าสู่สายงาน AI Data Engineer และ LLMOps

---

## 1. กลุ่มเป้าหมายและข้อกำหนดเบื้องต้น (Audience & Prerequisites)

*   **ผู้เรียนเป้าหมาย**: นักศึกษาระดับ ปวส. สาขาคอมพิวเตอร์/เทคโนโลยีสารสนเทศ หรือผู้ปฏิบัติงานทั่วไปที่ต้องการเปลี่ยนสายงาน
*   **ความรู้พื้นฐานขั้นต่ำ (Prerequisites)**:
    *   ความรู้พื้นฐานภาษา Python (การใช้ตัวแปร, ลูป, ฟังก์ชัน และ API Requests เบื้องต้น)
    *   ความเข้าใจเรื่องฐานข้อมูล (SQL หรือ JSON format) เบื้องต้น
    *   ความรู้พื้นฐานเกี่ยวกับ Docker (การพิมพ์คำสั่ง docker-compose up/down เบื้องต้น)

---

## 2. ผลลัพธ์การเรียนรู้ที่ตั้งเป้าหมาย (Learning Outcomes)

เมื่อจบหลักสูตรนี้ ผู้เรียนจะสามารถ:
1.  **LO-1**: อธิบายบทบาทหน้าที่ของ Data Engineer และอธิบายการทำงานพื้นฐานของ Apache Airflow ในฐานะเครื่องมือจัดสัดส่วนงาน (Workflow Orchestrator) ได้
2.  **LO-2**: เข้าใจความหมายและสถาปัตยกรรมของ RAG (Retrieval-Augmented Generation) สามารถทำการแบ่งข้อความ (Chunking) และแปลงรูปข้อมูลตัวอักษรเป็นเวกเตอร์ (Embedding) ได้
3.  **LO-3**: สร้างและใช้งาน Vector Database (ChromaDB) เพื่อบันทึกข้อมูลแบบเวกเตอร์และสืบค้น (Query) ค้นหาข้อมูลที่ใกล้เคียงได้
4.  **LO-4**: พัฒนา Airflow DAG 3.x โดยใช้ฟังก์ชันระดับสูงของ TaskFlow API และ `@task.llm` ร่วมกับ Gemini เพื่อสร้างระบบ RAG อัตโนมัติเมื่อมีไฟล์ใหม่เข้ามา
5.  **LO-5**: พัฒนา AI Agentic pipeline ด้วย `@task.agent` ร่วมกับการทำ Human-in-the-loop เพื่อรอการพิจารณาตรวจสอบคำตอบจากผู้ใช้ก่อนจัดเก็บ
6.  **LO-6**: อธิบายกระบวนการทำงานแบบ MLOps และ DataOps ในการดูแลคุณภาพท่อส่งข้อมูล (Data Pipeline) และเวอร์ชันของ embeddings/models ได้

## 3. กำหนดการสอนรายชั่วโมง (Hourly Agenda)

### [09:00 – 10:00] Session 1: Foundation (1.0 ชั่วโมง - ทฤษฎี)
*   **เป้าหมาย**: ปูพื้นฐานบทบาท DE และ แนะนำ Airflow 3
*   **09:00 – 09:30**: บทบาทของ Data Engineer (ETL/ELT, Batch vs Streaming, Orchestration vs Scripting)
*   **09:30 – 10:00**: แนะนำ Apache Airflow 3 (สถาปัตยกรรม, UI, การออกแบบ DAG และ Task Instances)
*   **การประเมิน**: แบบทดสอบสั้นก่อนเรียน (Diagnostic Pre-check)

### [10:00 – 12:00] Session 2: RAG Architecture (2.0 ชั่วโมง - ทฤษฎี/ปฏิบัติ)
*   **เป้าหมาย**: เข้าใจทฤษฎี RAG และพัฒนา Lab แรกด้วย Jupyter & Gemini
*   **10:00 – 10:45**: ทฤษฎี RAG (ความหมาย, Chunking, Embedding, Vector DB) และการเปรียบเทียบกับโมเดลแบบ Long-context
*   **10:45 – 12:00**: **Lab 02 (Jupyter Notebook)**: พัฒนา RAG ทีละขั้น ตั้งแต่การอ่านเอกสาร, แบ่งข้อความ, เชื่อมต่อ Gemini Embedding, นำเข้า ChromaDB และทำการ Query ค้นหาแหล่งที่มา
*   **การประเมิน**: แบบทดสอบความเข้าใจหลังโมดูลและงานเขียนโปรแกรมสืบค้น

### [12:00 – 13:00] พักกลางวัน (1 ชั่วโมง)

### [13:00 – 14:30] Session 3: RAG via Airflow Operators (1.5 ชั่วโมง - ปฏิบัติ)
*   **เป้าหมาย**: แปลง Logic จาก Session 2 ให้กลายเป็น Automated Pipeline บน Airflow 3
*   **13:00 – 13:30**: แนะนำชุดตกแต่งงานรูปแบบใหม่ `@task.llm` และไลบรารี `apache-airflow-providers-common-ai`
*   **13:30 – 14:30**: **Lab 03 (Airflow 3)**: พัฒนา DAG ที่ตรวจจับไฟล์ใหม่ (ใช้ Sensor) จากนั้นประมวลผล RAG (Chunking -> Vector Database) และส่งคำตอบโดยใช้ Decorator LLM โดยอัตโนมัติ

### [14:30 – 15:30] Session 4: AI Agent Bonus (1.0 ชั่วโมง - ปฏิบัติ)
*   **เป้าหมาย**: อัปเกรด Pipeline ให้มีตรรกะคิดวิเคราะห์ (Agentic) และการทวนสอบโดยมนุษย์ (HITL)
*   **14:30 – 14:45**: ทฤษฎี AI Agent ในมุมของ Task Flow และแนวคิด Human-in-the-loop (HITL) ใน Airflow 3
*   **14:45 – 15:30**: **Lab 04 (Airflow 3)**: พัฒนา DAG ที่ใช้ `@task.agent` ร่วมกับ toolsets ในการค้นหาเอกสาร และการใส่ `HITLOperator` เพื่อหยุดท่อข้อมูลชั่วคราวและส่งแบบฟอร์มให้ผู้สอน/แอดมินยืนยันคำตอบผ่าน UI ก่อนบันทึกคำตอบสุดท้าย

### [15:30 – 16:00] Session 5: MLOps/DataOps & Wrap-up (0.5 ชั่วโมง - ทฤษฎี)
*   **เป้าหมาย**: สรุปบทเรียนภาพรวมการนำไปใช้งานจริง
*   **15:30 – 15:50**: กรอบความคิด DataOps และ MLOps (การติดตามการแปลงเวกเตอร์, การทดสอบความถูกต้องของข้อมูล, การสลับโมเดล)
*   **15:50 – 16:00**: ถามตอบ (Q&A) และกรอกแบบประเมินหลักสูตร
