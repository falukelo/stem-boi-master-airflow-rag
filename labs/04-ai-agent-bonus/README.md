# Lab 04: AI Agent & Human-in-the-Loop (HITL)

ยินดีต้อนรับเข้าสู่บทปฏิบัติการบทวิเคราะห์ขั้นสูง ในแล็บนี้เราจะมาเจาะลึกการใช้งาน **AI Agentic Workflows** บน **Apache Airflow 3.x** ผ่านฟังก์ชันระดับสูง `@task.agent` พร้อมทำการเปรียบเทียบความแตกต่างระหว่างสถาปัตยกรรม LLM แบบดั้งเดิมและเอเจนต์

## 1. เปรียบเทียบความแตกต่างเชิงสถาปัตยกรรม (Comparison)

ก่อนเริ่มเขียนโค้ด ผู้เรียนควรทำความเข้าใจความสอดคล้องและการนำไปใช้ที่แตกต่างกันดังนี้:

| คุณลักษณะ | `@task.llm` (RAG แบบปกติ - Lab 3) | `@task.agent` (Agentic RAG - Lab 4) |
| :--- | :--- | :--- |
| **โครงสร้างการคิด** | **รันรอบเดียวจบ (Single Step)**: นำบริบทท่อข้อมูลที่จัดเตรียมไว้แนบ Prompt ทันที | **คิดวิเคราะห์ตามลูป (Multi-step Reasoning)**: โมเดลมีกระบวนการวนลูปวิเคราะห์ความคิดด้วย ReAct Framework |
| **การเลือกใช้ข้อมูล** | **คงที่ (Static)**: นักพัฒนาต้องกำหนดและคัดลอก Chunk มาแนบไว้ให้เสร็จสิ้น | **ยืดหยุ่น (Dynamic)**: เอเจนต์ตัดสินใจเองว่าจะดึงเครื่องมือใดมาใช้งาน คิวรีกี่ครั้ง หรือคัดเฉพาะ ID ใด |
| **การเข้าถึงทรัพยากร** | จำกัดเฉพาะข้อมูลที่ยัดเข้าไปใน Prompt ตั้งแต่เริ่มต้น | เข้าถึงระบบภายนอกผ่านการต่อเชื่อมและเรียกใช้งาน **Toolsets** (เช่น ฐานข้อมูลเวกเตอร์ หรือ APIs ภายนอก) |
| **ความเหมาะสม** | เหมาะกับการตอบสล๊อตข้อมูลธรรมดา สรุปเอกสารเดี่ยว หรือคำถามเส้นตรง | เหมาะสำหรับคำถามที่ซับซ้อน ต้องการคิวรีหาความเชื่อมโยงของข้อมูลหลายแหล่ง หรือหาเหตุผลเปรียบเทียบ |

## 2. โครงสร้างแล็บ 4: เอเจนต์และมนุษย์ยืนยันตัวตน (`lab4b_task_agent_hitl`)

ในการทำงานจริงที่เกี่ยวข้องกับข้อมูลนโยบายบริษัทและสวัสดิการของพนักงาน เราต้องการให้มีเอเจนต์วิเคราะห์หาคำตอบจากฐานข้อมูล ChromaDB ผ่านเครื่องมือค้นหา แต่ก่อนที่จะบันทึกข้อความสรุปออกระบบหลัก เราต้องคอยกักระบบให้มนุษย์เข้ามาตรวจสอบ (HITL) เพื่อป้องกันการหลอนตัวเลขของเอเจนต์

### โครงสร้างการทำงานของ DAG:
1.  **`receive_user_query`**: รับข้อความคำถามจาก DAG run conf
2.  **`ai_agent_reasoning`** (`@task.agent`):
    *   ใช้โมเดล Gemini วิเคราะห์หาคำตอบ
    *   เรียกใช้เครื่องมือค้นหา `search_vector_db_tool` เพื่อดึงเนื้อหาจาก ChromaDB มาวิเคราะห์ประโยค
3.  **`wait_for_human_review`** (`HITLOperator`):
    *   สั่งเปลี่ยนสถานะ Task เป็น `awaiting_input` (หยุดพักการใช้ทรัพยากร CPU)
    *   ส่งรายละเอียดคำถามและคำตอบที่วิเคราะห์โดยเอเจนต์ขึ้นหน้าจอเว็บฟอร์ม
4.  **`save_final_response`**: จัดเก็บผลบันทึกคำตอบสุดท้ายที่พนักงานยืนยันอนุมัติแล้ว

## 3. ขั้นตอนการทดสอบรันแล็บ

*ตรวจสอบให้แน่ใจว่าได้คัดลอกสคริปต์ไปวางแล้ว: `cp labs/04-ai-agent-bonus/dags/dag_task_agent_hitl.py labs/03-rag-airflow/dags/`*

1.  เปิดการทำงาน DAG `lab4b_task_agent_hitl` ใน Airflow UI
2.  สั่งรันผ่านเมนู **Trigger w/ config** โดยป้อนคำถามจำลอง เช่น:
    ```json
    {
      "query": "บริษัทมีสวัสดิการเบิกค่ารักษาพยาบาลต่อปีสูงสุดเท่าไร และครอบคลุมค่าเบิกเคลมฟันได้กี่บาท?"
    }
    ```
3.  สังเกต DAG จะทำการสืบค้นเวกเตอร์ด้วยตนเอง และไปหยุดค้างที่ Task `wait_for_human_review`
4.  กดคลิกที่ชื่อ Task `wait_for_human_review` -> กดเลือกแท็บฟอร์มเมนู **HITL / Approval Request**
5.  ตรวจสอบคำตอบของเอเจนต์ (ซึ่งเอเจนต์ควรอ่านคิวรีข้อมูลมาตอบว่า รักษาพยาบาลปีละ 50,000 บาท และครอบคลุมค่าเบิกเคลมฟัน 5,000 บาท)
6.  กดปุ่ม **Approve** เพื่อยืนยันและปลดล็อกการหยุดพักให้ท่อข้อมูลบันทึกข้อความสรุปปลายทางเรียบร้อย
