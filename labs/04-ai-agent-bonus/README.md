# Lab 04: AI Agent Orchestration & Human-in-the-Loop (HITL)

ห้องปฏิบัติการโบนัสพิเศษ มุ่งเน้นการเปลี่ยนผ่านระบบท่อส่งข้อมูลแบบเส้นตรง ไปสู่การใช้ **AI Agentic Workflows** บน **Apache Airflow 3.x** ผ่านฟังก์ชัน `@task.agent` และผูกระบบควบคุมการตัดสินใจขั้นสูงสุดโดยให้มนุษย์เข้ามาตรวจสอบ (Human-in-the-Loop) ผ่านหน้าเว็บควบคุมของ Airflow ก่อนนำคำตอบไปใช้งานจริง

---

## 1. สิ่งที่คุณจะได้เรียนรู้ (Learning Objectives)
*   ความเข้าใจโครงสร้างการทำงานของ AI Agent (Reasoning -> Tools -> Observation)
*   การเขียนโปรแกรมระบุเครื่องมือ (Toolsets) ให้ Agent นำไปประยุกต์ค้นข้อมูลจาก Vector DB ด้วยตนเอง
*   การใช้งานระบบหยุดท่อส่งข้อมูลชั่วคราวโดยไม่ใช้กำลังทรัพยากรเซิร์ฟเวอร์ (`awaiting_input` state)
*   การใช้งานและตั้งค่า `HITLOperator` เพื่อสร้างฟอร์มอนุมัติคำตอบบนหน้า UI ของ Airflow 3.x

---

## 2. ขั้นตอนการตั้งค่าและการรันแล็บ (Running Instructions)

*หมายเหตุ: แล็บบทความนี้ใช้งานสภาพแวดล้อมรัน Docker ตัวเดียวกับ Lab 3 ดังนั้นตรวจสอบให้แน่ใจว่าระบบในโฟลเดอร์ `labs/03-rag-airflow` รันอยู่เรียบร้อยแล้ว*

### ขั้นตอนที่ 1: คัดลอก DAG เข้าสู่ระบบรันหลัก
ให้ทำการย้ายหรือก๊อปปี้ไฟล์ DAG ของเอเจนต์:
```bash
# คัดลอกไฟล์ DAG ไปยังไดเรกทอรีรันของ Airflow
cp labs/04-ai-agent-bonus/dags/agentic_rag.py labs/03-rag-airflow/dags/
```

### ขั้นตอนที่ 2: ตั้งค่าคำถามของพนักงาน (Trigger DAG with Configuration)
เนื่องจากเราต้องการทดสอบสอบถามข้อมูลที่มีความแตกต่างกันในชั้นเรียน ให้สั่งรัน DAG ผ่านเมนู **Trigger DAG -> Trigger DAG w/ config** ในหน้าจอ UI ของ Airflow:
1. ไปที่หน้าจอหลัก หา DAG ชื่อ `agentic_rag_with_hitl`
2. กดปุ่มลูกศรข้างขวาของปุ่มรัน เลือก **Trigger w/ config**
3. ป้อนข้อความคำถามในรูปแบบ JSON เช่น:
   ```json
   {
     "query": "ฉันเบิกเงินค่ารักษาพยาบาลได้สูงสุดปีละเท่าไร และสามารถยื่นเบิกได้ช่องทางไหน?"
   }
   ```
4. กดปุ่ม **Trigger**

### ขั้นตอนที่ 3: กระบวนการตรวจสอบโดยมนุษย์ (Human-in-the-Loop)
1. เมื่อเริ่มทำงาน DAG จะรันผ่านขั้นตอนการวิเคราะห์ของเอเจนต์ `ai_agent_reasoning` (โดยระบบจะเข้าไปคิวรีฐานข้อมูลเวกเตอร์ ChromaDB เพื่อเก็บข้อมูลประวัติสวัสดิการมาประมวลผลคำตอบอัตโนมัติ)
2. เมื่อผ่านขั้นตอนวิเคราะห์ DAG จะหยุดอยู่ที่ Task `wait_for_human_review` และแสดงสถานะสีส้ม/น้ำเงินอ่อน (**`awaiting_input`**)
3. ให้คุณกดเข้าไปที่ Task `wait_for_human_review` ในมุมมอง Grid/Graph view
4. กดที่หัวข้อฟอร์ม **HITL / Approval Request** บนหน้าต่างข้อมูลย่อย
5. คุณจะพบคำถามเดิมที่คุณตั้งค่าไว้ และพบช่อง **"คำตอบที่เอเจนต์คิดวิเคราะห์ขึ้นมา"**
6. ป้อนคำแนะนำปรับปรุง หรือยืนยันความถูกต้องผ่านปุ่ม **Approve** (หรือตรวจสอบแก้ไขข้อความก่อนส่งไปจัดเก็บปลายทาง)
7. เมื่อคุณกดยืนยันแล้ว DAG จะหลุดพ้นการหยุดพัก ดำเนินงานต่อไปยังขั้นตอนบันทึกข้อความสรุปเก็บถาวร (`save_final_response`) และเสร็จสิ้นโฟลว์งานทั้งหมดอย่างสมบูรณ์แบบ!
